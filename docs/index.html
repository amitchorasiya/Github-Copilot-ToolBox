Copilot Toolbox
Install in VS Code Source on GitHub

One place for Copilot-related setup.

Stop losing time to scattered rules, MCP config, and “skills” folders. Give your team a single, honest view inside VS Code.

In plain terms: Copilot only works as well as the setup around it—but that setup is usually scattered across files, machines, and habits. GitHub Copilot Toolbox is one dedicated Copilot Toolbox in VS Code: see what’s configured, standardize how teams move to Copilot (including from Cursor), and give Chat better context while each developer still chooses what to share.

For engineering teams

Faster path from Cursor to Copilot

Guided actions to port connections (MCP), rules, and scaffold a memory bank—without memorizing file formats.

Discover and add from one hub

Browse MCP registry and skills catalogs, see what’s installed, fewer raw JSON edits by hand.

One checklist view

Workspace vs personal setup, local skill folders, rules, instructions, memory bank—aligned with what you shipped.

Smarter context for Chat

Context packs and readiness flows with explicit choices so teams agree on what Copilot is allowed to see.

What you get in VS Code

Real UI from the extension—MCP & skills hub, Intelligence, Workspace kit.

Ready in the Marketplace

Install the extension, open Copilot Toolbox in the activity bar, then MCP & skills.

Install GitHub Copilot Toolbox

code --install-extension amitchorasiya.github-copilot-toolbox
